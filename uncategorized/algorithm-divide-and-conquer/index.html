Divide and Conquer

Posted on 2019-10-03

Strategy

The divide and conquer paradigm consists of three steps as follows.

  1. Divide: Divide the original problem with size \(n\) into the problem with smaller size.
  2. Conquer: Solve the problem with smaller size.
  3. Combine: Merge the result of the subproblem.

Maximum Subarray Problem

Definition

Given that an array \(A\) with length \(n\), find \(1 \leq i \leq j \leq n\) such that \(\sum_{m = i}^{j} A[m]\) is maximized.

Brute force

Enumerate all possible combination of \((i, j)\) and find the maximum \(\sum_{m = i}^{j} A[m]\) for each \(j\), with the time complexity \(\Omega(n^2)\).

Divide and Conquer

First, we shall divide the array \(A\) into two halves as \(A_1 = [1 .. mid]\) and \(A_2 = [mid + 1 .. n]\), where \(mid = \lceil \frac{n}{2} \rceil\).

Observe that if \(A[i .. j]\) is the maximum subarray of the \(A\), it has only three possibilities.

  1. \(A[i .. j]\) is the subarray of \(A_1\)
  2. \(A[i .. j]\) is the subarray of \(A_2\)
  3. \(A[i .. j]\) is crossing over \(A_1\) and \(A_2\).

For case 1 and case 2 are just the same problem with smaller problem size \(\lceil \frac{n}{2} \rceil\). The remaining problem is that how to find the maximum subarray crossing the midpoint.

This is another important observation for finding the maximum crossing subarray. The maximum crossing subarray must consists the maximum subarray in the form \(A[i .. mid]\) in \(A_1\) and \(A[mid .. j]\) in \(A_2\). Since the start / end index of the subarray is fixed as the middle point of \(A\). Therefore, the time complexity of finding crossing subarrays is O(n).

The complete algorithm is as follow.

  1. If the length of A is 1, return A.
  2. Divide the array A into two halved as \(A_1\) and \(A_2\).
  3. Find the maximum subarray of \(A_1\) and \(A_2\).
  4. Find the maximum crossing subarray of \(A\).
  5. Compare the sum of three subarrays and return the maximum.

The time complexity in recursive form is \(T(n) = 2 T(\lceil \frac{n}{2} \rceil) + \Omega (n)\). By master theorem, the explicit time complexity is \(\Omega(n \log n)\).

Matrix Multiplication

Strassen’s algorithm

Let the following variables. \[ P_1 = A_{11} (B_{12} - B_{22}) \\ P_2 = (A_{11} + A_{12}) B_{22} \\ P_3 = (A_{21} + A{22}) B_{11} \\ P_4 = A_{22} (B_{21} - B_{11}) \\ P_5 = (A_{11} + A_{22}) (B_{11} + B_{22}) \\ P_6 = (A_{12} - A_{22}) (B_{21} + B_{22}) \\ P_7 = (A_{11} - A_{21}) (B_{11} + B_{12}) \\ \]

Observe that, \[ C_{11} = P_4 + P_5 + P_6 - P_2 \\ C_{12} = P_1 + P_2 \\ C_{21} = P_3 + P_4 \\ C_{22} = P_1 + P_5 - P_3 - P_7 \\ \]

The time complexity in recursive form is \(T(n) = 7 T(\lceil \frac{n}{2} \rceil) + \Omega (n)\). By master theorem, the explicit time complexity is \(\Omega(n^{\log_2 7}) \approx \Omega(n ^ {2.81})\).
